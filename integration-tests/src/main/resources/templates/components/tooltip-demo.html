Tooltip

The PatternFly tooltip — a positioned bubble with an arrow, shown on hover/focus of a target. The runtime template renders just the bubble; positioning and trigger logic live with the consumer (either CSS positioning, browser-native title, or an Alpine wrapper).

Documentation

Tooltip props

Parameters accepted by {|{#include components/feedback/tooltip ... /}|}.

* required
Name Type Description
* id String DOM id — used as the aria-describedby target on the trigger.
* (content slot) Slot Tooltip text, passed as the include body. Keep it short — one line is ideal. Mirrors PF React's content prop.
position top | bottom | left | right Where the bubble sits relative to its container. Default top.
visible boolean Adds pf-m-visible — keeps the tooltip shown without a trigger. Used in demos.

Divergence from PF React. PF React's Tooltip wraps a trigger element and manages show/hide on hover/focus via JS. Our runtime only renders the bubble; positioning is done with CSS in the demo wrapper and show/hide should be wired by the consumer — either via the browser's built-in title attribute (simplest, no JS) or a small Alpine x-data component listening for mouseenter/focusin on the trigger.

Usage

Tooltips are a labelling tool, not a UI panel. Use them for short, single-line context — icon button labels, abbreviation expansions, full ISO timestamps. For longer help content use Popover.

Wire aria-describedby on the trigger. The trigger element should reference the tooltip's id via aria-describedby so AT consumers hear the tooltip text without needing to hover.
